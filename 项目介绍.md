# 在线投票系统 - 项目介绍

## 📖 项目概述

在线投票系统是一个基于Go语言开发的高性能、可扩展的投票平台，采用微服务架构设计，支持高并发访问和数据一致性保证。系统实现了完整的投票功能，包括创建投票、参与投票、查看结果等核心业务流程。

![系统架构图](https://via.placeholder.com/800x400?text=Microservices+Architecture)

## ✨ 核心特性

### 🚀 高性能
- **Go语言**：使用高性能Go语言开发
- **gRPC通信**：高效的RPC调用机制
- **Redis存储**：内存数据库，毫秒级响应
- **异步处理**：支持高并发访问

### 🛡️ 数据一致性
- **防重复投票**：基于用户ID的投票记录
- **原子操作**：使用Lua脚本保证数据一致性
- **幂等设计**：支持重试和撤销操作
- **状态控制**：投票开启/关闭状态管理

### 🌐 实时更新
- **SSE技术**：服务器推送事件实现实时更新
- **即时反馈**：投票结果即时显示
- **无轮询**：减少不必要的网络请求

### 📊 监控观测
- **Prometheus**：系统指标收集
- **Grafana**：可视化监控面板
- **健康检查**：服务状态监控
- **性能监控**：响应时间和吞吐量

### ☸️ 云原生
- **Docker容器化**：标准化部署
- **Kubernetes编排**：弹性伸缩
- **服务发现**：自动服务注册发现
- **负载均衡**：请求分发处理

### 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端层** | 原生JavaScript | 轻量级实现，无需框架依赖 |
| **接入层** | Gin + gRPC | HTTP网关，负载均衡，限流 |
| **业务层** | gRPC服务 | 投票核心逻辑，业务处理 |
| **存储层** | Redis Cluster | 高性能数据存储 |
| **部署层** | Docker + K8s | 容器化，云原生部署 |
| **监控层** | Prometheus + Grafana | 监控告警，可视化 |

## 🎯 功能特性

### 投票管理
- ✅ 创建投票：自定义问题、选项、有效期
- ✅ 关闭投票：管理员手动结束投票
- ✅ 删除投票：清理过期或无效投票
- ✅ 搜索投票：按创建者、时间等条件筛选

### 投票参与
- ✅ 投票功能：用户选择选项并投票
- ✅ 撤销投票：支持撤销已投的票
- ✅ 防重复：同一用户只能投票一次
- ✅ 实时更新：投票结果实时显示

### 数据展示
- ✅ 投票统计：实时显示各选项得票数
- ✅ 投票列表：展示所有可用投票
- ✅ 用户历史：查看个人投票记录
- ✅ 公开投票：支持公开和私有投票

## 🔧 核心组件

### 1. HTTP网关 (httpserver)
- **功能**：接收HTTP请求，转发到gRPC服务
- **技术**：Gin框架
- **特性**：
  - 路由管理
  - 请求限流
  - 用户认证
  - 健康检查

### 2. 业务服务 (grpcserver)
- **功能**：处理投票核心业务逻辑
- **技术**：gRPC
- **特性**：
  - 投票创建/管理
  - 投票处理
  - 数据一致性
  - 状态管理

### 3. 数据存储 (Redis)
- **功能**：存储投票数据
- **技术**：Redis Cluster
- **数据结构**：
  - Hash：投票元数据
  - Set：投票记录
  - String：统计信息

### 4. 监控系统
- **Prometheus**：收集系统指标
- **Grafana**：可视化展示
- **监控指标**：
  - QPS/响应时间
  - 错误率
  - 资源使用
